Layout

Semantics

Semantic markup should be used to structure layouts. The main semantic components in Fluid are:

  • Left navbar - contains primary branding and navigation; represented by a <nav class="navbar"> tag
  • Top nav - contains secondary branding and navigation (such as breadcrumbs) and may contain other application level components (i.e. current user info, notifications, etc.); represented by a <nav class="fixed-top"> tag that is a direct child of the <body>
  • Main content - contains all other application content; represented by the <main> tag
    • Sections - use to divide major content areas; represented by <section> tags
  • Main footer - may contain tertiary branding/navigation and copyright/legal information, etc.; represented by the last <footer> tag that is a direct child of the <body>

Responsiveness

FLUID is designed for large screens. Therefore, the minimum width of the entire layout is limited by setting min-width: 1024px on the <body>, below which a horizontal scrollbar will appear. The maximum width of .container-fluid has been set to 1920px, above which the container will have equal left and right margins to fill up the remaining space. Generally, you should use .container-fluid so that layouts scale fluidly between these extremes. (The default breakpoints for .container have not been changed.)

The main <header class=".header-top"> and <footer class=".footer-bottom"> should not be placed inside any container so that they can stretch to the full width of the window.

Observe the behavior of the example containers below and the header/footer on this page as you change the width of the window to understand the desired behavior.

Navbar

The navbar on the left side of this demo is an example of the Fluid Bootstrap navbar. It uses standard Bootstrap markup and classes, but also demonstrates the Fluid features described below. Please examine the markup to learn the correct way to construct these features.

Brand

Custom content
Bootstrap does not make any assumptions about what goes in the brand area. If you use only text, Fluid Bootstrap will style it appropriately in expanded mode, but it will be clipped when the navbar is collapsed. Fluid prefers that you use an icon followed by text (although the font size and number of lines is not specified). In that case, Fluid Bootstrap ensures that the icon/text combination is displayed correctly in both expanded and collapsed modes.
Dropdown menu
A green arrow indicates that the brand can be clicked to open a dropdown menu listing other available products. This menu is also accessible when the navbar is collapsed.

Nav Items

Icons
Place an icon as the first child inside a nav item and it will be styled appropriately.
Dropdown menu
Three vertical dots indicate that there is a dropdown menu associated with a nav item. This menu is not accessible when the navbar is collapsed.
Child nav items
A nav item can have child nav items, indicated by an arrow on the right. These items are accessible when the navbar is collapsed by popping out to the right when their parent is hovered over. Note: This feature makes use of the Bootstrap collapse functionality. Correct positioning of these popovers requires some additional Javascript.
Status indicators
Fluid status indicators can be applied to nav items and they will be positioned appropriately.
